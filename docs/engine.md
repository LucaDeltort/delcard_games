# Game Engine Guide

This guide explains how to add a new card game to the platform. No framework knowledge is required — a game is just a TypeScript object that satisfies the `GameDefinition<S>` interface.

---

## Core concepts

### Cards

A `Card` has four fields:

```typescript
interface Card {
	id: string; // unique identifier (auto-generated by createCard)
	face: string; // e.g. 'A', 'K', '7', 'Joker'
	suit?: string; // e.g. 'spades', 'hearts' — omit for suitless cards
	isHidden: boolean;
}
```

`isHidden` is a hint to the UI: when `true`, only the owning player should see the face. The engine itself does not enforce visibility.

### Zones

A `GameZone` is any collection of cards — a deck, a hand, a discard pile, a player's score area, etc.

```typescript
interface GameZone {
	id: string;
	type: 'hidden' | 'public' | 'fan'; // display hint for the UI
	cards: Card[];
	ownerId?: string; // set when the zone belongs to a specific player
}
```

All card collections in a game live in `state.zones`, a flat `Record<string, GameZone>`. Name zones however you like — `'draw'`, `'discard'`, `'hand_alice'`, `'shield_bob'`.

### State

Every game state extends `GameStateGeneric`:

```typescript
interface GameStateGeneric {
	players: string[]; // player IDs, in seat order
	zones: Record<string, GameZone>;
	turnPlayerId: string;
	phase: string; // e.g. 'playing', 'scoring'
	activeGameId: string; // must match GameDefinition.id
}
```

Extend it with whatever extra fields your game needs:

```typescript
type MyGameState = GameStateGeneric & {
	phase: 'playing' | 'gameover';
	score: Record<string, number>;
};
```

### Actions

An `Action` is a player's move:

```typescript
type Action = {
	type: string;
	playerId: string;
	payload?: unknown; // cast inside applyAction
};
```

## The GameDefinition interface

```typescript
type DeckType = 'FrenchDeckWithJoker' | 'FrenchDeckWithoutJoker' | 'ColorDeck';

type GameDefinition<S extends GameStateGeneric> = {
	id: string;
	name: string;
	deckType: DeckType;
	minPlayers: number;
	maxPlayers: number;

	setup: (players: string[]) => S;
	getValidActions: (state: S, playerId: string) => Action[];
	applyAction: (state: S, action: Action) => S;
	isOver: (state: S) => boolean;
	getWinner: (state: S) => string | null;

	// Optional. Called by the host when a non-host player disconnects mid-game.
	// Return a state that continues (player removed/skipped) or ends the game.
	// Omit to fall back to a generic gameover when players drop below minPlayers.
	onPlayerDisconnect?: (state: S, playerId: string) => S;
};
```

**All methods are pure.** They must return new objects — never mutate `state`.

## Walkthrough: War

War is the simplest game in the repository. Reading its source alongside this walkthrough is the fastest way to understand the engine.

Source: [`src/lib/games/war.ts`](../src/lib/games/war.ts)

### 1 — Define the state type

```typescript
type WarState = GameStateGeneric & {
	phase: 'playing' | 'gameover';
};
```

War only needs one extra field beyond the base.

### 2 — Plan the zones

Each player needs three zones:

| Zone           | Type     | Contents                     |
| -------------- | -------- | ---------------------------- |
| `deck_<pid>`   | `hidden` | The player's remaining cards |
| `played_<pid>` | `public` | The card they just revealed  |
| `won_<pid>`    | `hidden` | Cards they've won            |

### 3 — Implement setup

```typescript
setup(players) {
  const { hands } = deal(createDeck('FrenchDeckWithoutJoker'), 26, 2);
  const zones: WarState['zones'] = {};
  players.forEach((pid, i) => {
    zones[`deck_${pid}`]   = createZone(`deck_${pid}`,   'hidden', hands[i], pid);
    zones[`played_${pid}`] = createZone(`played_${pid}`, 'public', [],       pid);
    zones[`won_${pid}`]    = createZone(`won_${pid}`,    'hidden', [],       pid);
  });
  return { players, zones, turnPlayerId: players[0], phase: 'playing', activeGameId: 'war' };
}
```

Note: Pass `deckType` to `createDeck()`. War uses `'FrenchDeckWithoutJoker'` (52 cards). See the Tips section below for all available deck types.

### 4 — Implement getValidActions

Return an empty array when the player has no legal moves (including when it is not their turn).

```typescript
getValidActions(state, playerId) {
  if (state.phase !== 'playing') return [];
  if (state.turnPlayerId !== playerId) return [];
  if (state.zones[`deck_${playerId}`].cards.length === 0) return [];
  return [{ type: 'REVEAL', playerId }];
}
```

### 5 — Implement applyAction

`applyAction` is where the game logic lives. It must return a new state — never mutate the argument.

```typescript
applyAction(state, action) {
  if (action.type !== 'REVEAL') return state;
  // move top card from deck to played zone…
  // if both players have played, compare and resolve the round…
  // check win condition…
}
```

Use `moveCard` to move cards between zones — it handles the immutable update for you.

### 6 — Implement isOver and getWinner

```typescript
isOver(state)   { return state.phase === 'gameover'; }
getWinner(state) {
  if (state.phase !== 'gameover') return null;
  return state.players.reduce((best, p) => {
    const total = state.zones[`deck_${p}`].cards.length + state.zones[`won_${p}`].cards.length;
    const bestTotal = state.zones[`deck_${best}`].cards.length + state.zones[`won_${best}`].cards.length;
    return total > bestTotal ? p : best;
  });
}
```

## Engine utilities

All utilities are exported from `$lib/engine`.

### Cards

| Function                                | Description                                          |
| --------------------------------------- | ---------------------------------------------------- |
| `createCard(face, suit?, isHidden?)`    | Creates a card with a unique id                      |
| `shuffle(items)`                        | Fisher-Yates shuffle, returns a new array            |
| `deal(deck, countPerPlayer, nbPlayers)` | Round-robin deal, returns `{ hands, remaining }`     |
| `drawCard(deck)`                        | Returns `{ card, remaining }` or `null` if empty     |
| `topCard(pile)`                         | Peeks at the top card without removing it            |

### Zones

| Function                                 | Description                                           |
| ---------------------------------------- | ----------------------------------------------------- |
| `createZone(id, type, cards?, ownerId?)` | Creates a new zone                                    |
| `moveCard(zones, fromId, toId, cardId)`  | Moves a card between zones, returns updated zones map |

### Decks

| Function               | Description                      |
| ---------------------- | -------------------------------- |
| `createDeck(deckType)` | Shuffled deck for the given type |

### Turn

| Function                         | Description                                 |
| -------------------------------- | ------------------------------------------- |
| `nextPlayer(players, currentId)` | Returns the next player ID, wrapping around |

## Tips

**Deck types.** Built-in options:
- `'FrenchDeckWithJoker'` — 54 cards (52 + 2 jokers)
- `'FrenchDeckWithoutJoker'` — 52 standard cards
- `'ColorDeck'` — 108 Color cards (colored numbers, action cards, wilds)

To add a completely new deck type, see [card-pack.md](card-pack.md#creating-a-new-deck-type).

**Custom one-off cards.** If your game needs a small number of game-specific cards on top of an existing deck, create them locally in `setup` with `createCard` and add them to the zone directly.

**Tracking non-card state.** HP, scores, timers — store them directly on the state object as extra fields. Only use zones for actual cards.

**Immutability.** Spread objects when modifying state: `{ ...state, phase: 'gameover' }`. The engine pattern relies on referential equality checks.

## Full integration flow

Once your `GameDefinition` is implemented, four more steps wire it into the app.

---

### 1 — Register the game

Edit `src/lib/games/index.ts`.

**Add the import and map entry:**

```typescript
import { myGame } from './my-game'

export const games = { war, fight, myGame } as unknown as Record<string, GameDefinition<GameStateGeneric>>
```

**Add the rules strings** (shown in the in-app rules drawer):

```typescript
export const gameRules: Record<string, { en: string; fr: string }> = {
  // …existing entries…
  myGame: {
    en: `2–4 players · 52 cards\n\n• Rule one.\n• Rule two.`,
    fr: `2–4 joueurs · 52 cartes\n\n• Règle un.\n• Règle deux.`
  }
}
```

---

### 2 — Add i18n strings

Edit `src/lib/i18n/en.ts` and `src/lib/i18n/fr.ts`.

At minimum, add a `name` key — used in the lobby header and home screen:

```typescript
// en.ts
myGame: {
  name: 'My Game',
  // add action labels, zone names, etc. as needed
}
```

```typescript
// fr.ts
myGame: {
  name: 'Mon Jeu',
}
```

In components, read it with `$t('myGame.name')`.

---

### 3 — Build a view (optional)

A generic fallback in `src/routes/game/[id]/+page.svelte` renders any game automatically — it shows all zones as card grids and all valid actions as buttons. You can ship and play without a custom view.

When you want a polished UI, create `src/lib/components/games/MyGameView.svelte`. All game views share the same props contract:

```typescript
let {
  state,
  myPlayerId,
  players,
  validActions,
  onAction
}: {
  state: GameStateGeneric          // cast to your state type inside
  myPlayerId: string
  players: LobbyPlayer[]           // { id, name } — use name for display
  validActions: Action[]
  onAction: (action: Action) => void
} = $props()
```

See `WarView.svelte` for a minimal example, `FightView.svelte` for a complex one.

---

### 4 — Wire the view in the route

Once you have a view component, add a branch in `src/routes/game/[id]/+page.svelte` — right before the generic fallback block (the `{:else}` near the bottom):

```svelte
<!-- at top of <script> -->
import MyGameView from '$lib/components/games/MyGameView.svelte'

<!-- in the template, before the generic {:else} -->
{:else if gameState.activeGameId === 'myGame'}
  <MyGameView
    state={gameState}
    {myPlayerId}
    players={enrichedPlayers}
    {validActions}
    onAction={submitAction}
  />
```

The `activeGameId` string must match the `id` field on your `GameDefinition`.

**Game over state:** When `state.phase === 'gameover'`, the game room container renders a fixed top banner (slide-in from top) showing the winner's name via `getWinner` and action buttons (rematch for host, back home). The game view stays mounted underneath, frozen with empty `validActions`. Game views do **not** need to handle the gameover phase themselves.
